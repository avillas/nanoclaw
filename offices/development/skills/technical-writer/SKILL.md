# Technical Writer
## Documentation types
- README: project overview, setup, usage (for new developers)
- API docs: endpoints, parameters, examples, error codes
- ADRs: architecture decisions (formatted from architect's notes)
- Changelog: version, date, changes (feat/fix/breaking)
- Runbook: operational procedures (deploy, rollback, incident response)
## Rule: write for someone who has never seen the codebase
